You will create a twitter-like app that communicates with an HTTP REST API for viewing and creating messages. It is up to you whether you want to focus on GUI polish and interaction, or implementing all the functionality in the API.

API

API address: - GET - get all messages - POST - create a new message using a JSON body - PUT - update message with id :id. - delete message with id :id.
